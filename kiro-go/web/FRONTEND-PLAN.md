# PivotStack 前端改造计划

## 现状摘要

- 单文件 SPA：`web/index.html`，2667 行，HTML + CSS + JS 全部内联
- 无框架，原生 JS
- 主色紫色 `#7c3aed`，浅色主题
- 三个 Tab：账号管理 / 设置 / API 端点

---

## 一、工程化拆分（优先级：高）

**问题**：2667 行堆在一个文件里，改动困难，样式/逻辑/结构混杂。

**方案**：不引入构建工具，保持 Go 直接 embed 的方式，但拆成多文件通过 Go 的 `//go:embed` 一起打包：

```
web/
├── index.html          # 只保留骨架 + 引用
├── css/
│   ├── base.css        # reset、变量、基础排版
│   ├── layout.css      # 容器、header、tabs
│   ├── components.css  # button、card、modal、badge、input
│   └── pages.css       # 各页面专属样式
└── js/
    ├── api.js          # 所有 fetch 封装
    ├── accounts.js     # 账号列表、CRUD、批量操作
    ├── settings.js     # 设置页逻辑
    ├── auth.js         # 登录/登出/密码缓存
    └── app.js          # 入口、路由、i18n
```

Go 侧：
```go
//go:embed web/*
var webFS embed.FS
```

**收益**：每个文件 < 300 行，可读性大幅提升，团队协作不冲突。

## 三、功能增强（优先级：中）

### 3.1 实时刷新

轮询 `/admin/api/accounts` 和 `/admin/api/stats`，每 30 秒自动更新，不需要手动刷新页面。状态变化时高亮闪烁提示。

### 3.2 账号筛选增强

现在只有搜索框，增加：
- 状态筛选：全部 / 启用 / 禁用 / 封禁 / 即将到期
- 区域筛选：ap-east / us-east 等
- 排序：按配额用量、请求数、到期时间

筛选状态存 localStorage，刷新后保持。

### 3.3 批量操作优化

现在批量操作是顶部按钮，改为：
- 列表左侧加 checkbox
- 底部弹出操作栏（仿 Gmail 样式）：已选 N 个 · [启用] [停用] [刷新] [删除]

### 3.4 账号健康度指示

在每个账号旁边显示一个彩色圆点：
- 🟢 绿：配额充足 + Token 有效
- 🟡 黄：配额 > 70% 或 Token 7 天内到期
- 🔴 红：配额耗尽 / Token 过期 / 被封禁

### 3.5 请求日志面板

新增第四个 Tab「日志」，展示最近 100 条请求：
- 时间、模型、账号、耗时、Token 数、状态
- 可筛选成功/失败
- 失败条目点开看错误详情

## 四、交互体验（优先级：中）

### 4.1 Toast 通知

用统一的 toast 替代 alert()：
- 右下角浮层
- 3 秒自动消失
- 支持 success / error / warning / info 四种

### 4.2 骨架屏

数据加载时用骨架屏占位，替代现在的空列表或 loading 文字。

### 4.3 确认框优化

删除账号等危险操作弹出自定义确认框，而不是 `confirm()`，支持输入账号邮箱二次确认。

### 4.4 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `?` | 显示快捷键帮助 |
| `Ctrl+F` | 聚焦搜索框 |
| `Ctrl+R` | 刷新所有账号 |
| `Escape` | 关闭当前 Modal |
| `1/2/3/4` | 切换 Tab |

## 技术约束

- **不引入 npm/构建工具**：保持 Go embed 直接打包，零部署依赖
- **不用重型框架**：保持原生 JS 或最多引入 Alpine.js（4KB gzip）做响应式
- **向后兼容**：所有 API 接口不变，只改前端
